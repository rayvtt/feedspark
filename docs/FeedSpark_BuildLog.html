Build Log

One control surface over every Claude Code session: what's queued, in build, shipped & live, dropped — and where sessions overlap. Shipped/in-build/dropped derive automatically from the repo's PRs and branches (they can't go stale); only the queue is yours to curate.

—

Queue — not built yet

Your backlog: add what you want built; a session picks it up and it moves right. Saved shared (concurrency-safe KV) — everyone sees one queue.

In build — sessions working now

Open PRs + active claude/* branches that haven't opened a PR yet.

Overlap radar

Active branches touching the same files — 🔥 means a hot file (worker.js, wrangler.toml, CLAUDE.md, app-page monoliths): sequence, don't parallel-edit.

Shipped

Merged PRs, newest first. LIVE = at or before the build currently serving the FCC (/api/version).

Not built — closed without merging / dropped from queue

Explicit decisions, kept visible so they aren't re-litigated by accident.
