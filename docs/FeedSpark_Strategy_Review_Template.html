Agentic commerce

Three protocols that let AI agents shop on behalf of consumers

These are emerging open standards. None requires immediate action from [Client] — but they explain why the attribute build matters - the agents need structured data to act.

UCP — Universal Commerce Protocol

Google, Shopify, Target, Walmart

Common language for AI commerce
  • Enables agents to handle discounts, loyalty, subscriptions
  • Complete checkout inside the AI conversation
  • Activated via native_commerce flag in Merchant Center
  • Relevant now: [client mechanic] maps here
MCP — Model Context Protocol

Anthropic, adopted by Shopify

Real-time data access for AI assistants
  • Live inventory, pricing, order status via API
  • Shopify merchants get automatic MCP endpoints
  • Any AI assistant speaking MCP can access store data
  • Relevant later: depends on [Client]'s platform architecture
ACP — Agentic Commerce Protocol

OpenAI (powers ChatGPT Shopping)

Instant checkout inside ChatGPT
  • Integrates with Stripe, PayPal
  • Users complete purchases without leaving the conversation
  • Currently US-only for checkout
  • Relevant now: discovery is global
    even if checkout isn't

The through-line across all three

Every protocol assumes the agent can read structured product data — attributes, Q&A, related products, variant options.
The feed isn't just a shopping ad input anymore. It's the API that AI agents call to reason about your product.

The Google Shopping → ChatGPT pipeline

A March 2026 study of 43,000 ChatGPT shopping carousel products found that 83% match Google Shopping's top 40 organic listings, with 60% from the top 10. This means optimising the Google Merchant Center feed is simultaneously optimising for ChatGPT visibility.

Theories

How conversational attributes could compound across surfaces

None of these are proven —
They're hypotheses drawn from the data above, each one testable within the programme.

Theory 1: Q&A → AI citation loop

If question_and_answer provides the AI agent with a pre-structured answer, the agent can cite [Client] directly rather than inferring from a description. Cited brands get 35% more organic clicks.

The compound effect: more clicks → better Shopping Graph signal → higher organic Shopping rank → more AI citations.

Test: Measure AI Overview citation rate for SKUs with Q&A vs without, using AI visibility tracking (Profound or Otterly AI).

Theory 2: Variant options → range capture

When a shopper asks "do you do this in [a bigger size / another colour]?", the agent needs variant_option to answer without sending them away. Clean variant data lets the agent recommend across the range in one answer.

Test: Compare cross-sell rate for products with item_group_title + variant_option vs products without.

Theory 3: Popularity rank → agent confidence

AI agents use ranking signals to break ties. popularity_rank is a direct, first-party signal that says "this is our best-seller." Ranked products carry higher citation confidence than unranked competitors.

Test: A/B test impression share for SKUs with popularity_rank populated vs control group.

Theory 4: Document links → authority signal

document_link pointing to spec sheets, certifications, or size guides gives the agent something to cite as authoritative. Google's Shopping Graph already cross-references structured data.

Test: Monitor whether Google AI Mode cites [Client]'s linked documents.

Theory 5: Description richness → long-tail capture

Descriptions under 200 chars match almost nothing in long-tail. At 500+ chars with use-case language, the description becomes the semantic field the agent matches against.

Test: Measure impression volume for 8+ word queries before and after description expansion.

Theory 6: Feed completeness → PMax efficiency

PMax campaigns depend heavily on feed quality. The shift from keyword-matching to attribute-matching means every unpopulated field is impression share left on the table.

Test: Track PMax performance metrics (CPA, ROAS) as attribute completeness increases.

The testing advantage: Google's product data experiments now support controlled A/B testing on feed attributes — including in PMax.

Why this matters for AI discovery

Product images in the shopping feed are no longer shown only in PLAs and Free Listings — they also appear in ChatGPT Shopping and Google AI Mode. But there's a catch:

Overlays don't render in ChatGPT. An apparel brand ran a PPC overlay and saw +16% CTR and +33% conversions — but only in Google Shopping. In ChatGPT, the agent reads the feed data behind the image, not the image itself.

The visual harvest closes this gap: attributes Tachyon reads from the image get written back into the feed as structured data — making the product discoverable even when the image isn't displayed.

The measurement gap

Traditional search console data doesn't capture AI citations. The baseline test addresses this: we need a tool (Profound, Otterly AI, or manual tracking) to measure how often [Client] appears in ChatGPT recommendations, Google AI Mode answers, and Perplexity product cards.

This is the single biggest risk to the programme — not that the attribute build won't work, but that we won't be able to prove it worked. Recommending budget for AI visibility tracking alongside the attribute build.

The gap is the opportunity. Every attribute [Client] populates that competitors don't is a structural advantage that compounds as AI surfaces grow.
